        Accessibility Notes

        • ARIA Combobox Pattern: Uses role="combobox" with aria-expanded, aria-haspopup="listbox", and aria-controls
        • Labels: Always pair combobox with visible labels using for and id attributes
        • Chips: Use role="list" on container and role="listitem" on each chip for screen readers
        • Remove Buttons: Each chip remove button needs descriptive aria-label (e.g., "Remove Ruby")
        • Error States: Use aria-invalid="true" and aria-describedby to link error messages
        • Focus State: 3px purple outline (#c3bde5) with 1px offset on container — CRITICAL for accessibility
        • Keyboard Navigation:
          • Tab to focus combobox
          • Type to filter options
          • Arrow Down/Up to navigate options
          • Enter/Space to select option
          • Escape to close dropdown
          • Backspace to remove last chip
        • Hidden Select: Native <select multiple> with hidden and aria-hidden="true" for form submission
        • Live Regions: Consider adding aria-live for dynamic updates when chips are added/removed

        Usage Guidelines

        • Multi-Select Only: This combobox is designed for multi-select use cases with chips
        • Searchable: Users can type to filter options — always provide meaningful search functionality
        • Clear All: Show clear button when chips are present for quick reset
        • Chip Management: Users can remove individual chips via × button or backspace key
        • Dropdown Toggle: Clicking toggle or focusing input should open dropdown
        • Form Submission: Use hidden <select multiple> for native form submission support
        • Placeholder: Change placeholder text when chips are present (e.g., "Search..." → "Add more...")
        • No Results: Show "No results found" message when search returns empty
        • Size Selection: Use --sm for compact layouts, --lg for better touch targets on mobile

        Implementation Notes

        This component requires JavaScript for full functionality. Agent B will add interactive behavior in the next step.

        • Data Attribute: Use data-combobox to initialize component instances
        • Hidden Select: Populate with available options — JavaScript syncs selected state
        • Chips: Dynamically generated based on selected options in hidden select
        • Dropdown Options: Dynamically populated from hidden select options
        • Search Filter: JavaScript filters dropdown options based on input value
        • Keyboard Support: Full keyboard navigation for accessibility compliance
