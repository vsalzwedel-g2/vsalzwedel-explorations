Accessibility Notes

  • Labels: Always pair selects with labels using for and id attributes
  • ARIA Labels: Include aria-label when label is not visible
  • Error States: Use aria-invalid="true" and aria-describedby for error messages
  • Required Fields: Use required attribute and aria-required="true"
  • Focus State: 3px solid outline (#c3bde5 / P40) — CRITICAL for accessibility
  • Keyboard Navigation:
    • Tab to focus select
    • Space or Enter to open dropdown
    • Arrow Up/Down to navigate options
    • Enter to select
    • Escape to close without selecting
  • Placeholder Option: Use disabled selected for placeholder text
  • Helper Text: Use aria-describedby to link helper text to select
  • Touch Targets: Minimum 40px height, use 48px for better mobile accessibility
  • Native Functionality: Uses native <select> for maximum compatibility and accessibility

Implementation Notes

  • Native Select: Uses native HTML <select> for maximum compatibility, accessibility, and mobile OS integration
  • No JavaScript Required: Fully functional without JavaScript — keyboard and mouse interactions work natively
  • Custom Styling: Browser default styling removed with appearance: none for consistent cross-browser appearance
  • Dropdown Icon: Custom icon positioned absolutely to prevent click-blocking — uses pointer-events: none
  • Placeholder Pattern: First option with disabled selected attributes creates placeholder text effect
  • Grouped Options: Use <optgroup> for categorizing related options
  • Mobile Behavior: Native select on mobile devices shows OS-native picker for optimal UX
