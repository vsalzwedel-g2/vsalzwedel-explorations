<script>
function updateCount(textarea) {
  const count = textarea.value.length;
  const max = textarea.maxLength;
  const counter = textarea.nextElementSibling;
  counter.textContent = `${count} / ${max}`;
}
</script>

Accessibility Notes

  • Labels: Always pair with a visible label using <label for="id">
  • ARIA Labels: Include aria-label when no visible label exists
  • Required Fields: Use required attribute and aria-required="true"
  • Error States: Use aria-invalid="true" and aria-describedby to link error messages
  • Error Messages: Include role="alert" on error text for screen readers
  • Focus State: 3px light purple outline (#c3bde5) with 1px offset — CRITICAL for accessibility
  • Keyboard: Tab to navigate, Shift+Tab to go back
  • Resize: Vertical only (accessible to keyboard users via browser controls)
  • Character Count: Use maxlength to enforce limits, counter is visual only
  • Placeholder: Never use placeholder as the only label (accessibility issue)

Usage Guidelines

  • When to Use: For multi-line text input such as comments, descriptions, messages, or feedback
  • Labels: Always include a visible label above the textarea
  • Helper Text: Provide guidance on what to enter or format requirements
  • Character Limits: Use character count for fields with length limits (e.g., reviews, bios)
  • Height: Default min-height (80px) works for most cases; increase for longer content
  • Resize: Vertical resize allows users to adjust height as needed
  • Required Fields: Mark with red asterisk (*) and include in label
  • Error Validation: Show errors below the field with specific, actionable messages
  • Disabled State: Use sparingly, only when field truly cannot be edited
