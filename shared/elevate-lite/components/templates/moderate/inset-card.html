Usage Guidelines

  • Purpose: Container for secondary, nested, or supporting content
  • Visual Hierarchy: Inset cards are less prominent than standard cards
  • Nesting: Common to place inset cards inside standard cards
  • Background: Default uses bg-neutral-5 for subtle recessed appearance
  • vs Standard Card: Standard cards float on page; inset cards are embedded within
  • Common Uses: Quotes, code blocks, notes, warnings, nested forms
  • Accent Borders: Can add left border for warnings/notes (4px solid)
  • Size: Use small for compact info, large for emphasis

Accessibility Notes

  • Semantic HTML: Use appropriate semantic tags for content (blockquote, code, aside)
  • Color Contrast: All text meets WCAG AA standards on default backgrounds
  • Focus: Interactive elements inside inset cards should have visible focus
  • Landmarks: For important notes, consider <aside> with aria-label
  • Visual Only: Don't rely on background color alone to convey meaning

Standard Card vs Inset Card

Understanding when to use each card type

Aspect Standard Card Inset Card
Background var(--bg-neutral-0) / #ffffff (brightest) var(--bg-neutral-5) / #fafafa (recessed)
Elevation Floats on page Embedded within
Shadow Optional (floating only) Never
Usage Primary content containers Secondary nested content
Border Radius 12-16px (md-lg) 8px (sm)
