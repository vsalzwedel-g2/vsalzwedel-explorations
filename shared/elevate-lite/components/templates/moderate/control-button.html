Control Button

Reference: Elevate Lookbook | Specs: DESIGN.md

Overview

Control buttons are segmented button groups used in toolbars, control panels, and filtering interfaces. They can be icon-only or include text labels, and support left/right icons with optional dropdown menus.

  • Segmented groups — Buttons join together with caps: left_cap, middle, right_cap, none
  • 4 sizes — xs (24px), sm (32px), md (40px), lg (48px)
  • Icon + label support — Can be icon-only or have left/right icons with labels
  • Active states — Toggle-style highlighting for selected options
  • Dropdown menus — Optional dropdown indicator with divider

Usage Guidelines

  • Segmented groups — Use left_cap, middle, and right_cap to create joined button groups
  • Active states — Add is-active class and aria-pressed="true" for toggle behavior
  • Icon sizes — XS/SM buttons use 16px icons, MD/LG use 20px icons
  • Label sizing — XS/SM buttons use 13px labels, MD/LG use 14px labels
  • Accessibility — Always include aria-label for icon-only buttons, aria-pressed for toggles
  • Use cases — Toolbars, view switchers, text alignment, filtering controls, sorting options
  • Hover states — Background changes to light purple (bg-primary-20), text/fill changes to primary
  • Focus states — 3px purple outline with subtle shadow for keyboard navigation
  • Dropdown indicator — Use divider + caret icon for buttons with dropdown menus
  • Standalone vs. grouped — Use segment="none" for standalone, caps/middle for groups
