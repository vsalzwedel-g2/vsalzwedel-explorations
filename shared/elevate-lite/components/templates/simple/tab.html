Usage Guidelines

  • Navigation: Use tabs to organize related content into separate views within the same context
  • Button Element: Always use <button> for tab triggers (not links or divs)
  • Active State: Exactly one tab must be active at all times
  • Content Relationship: Tab content should be closely related (settings sections, product details)
  • Tab Count: Keep 3-7 tabs; use different navigation pattern for more options
  • Icons: Use icons sparingly and only when they add clear meaning
  • Full Width: Use full-width variant when you have 2-4 tabs of similar label length

Accessibility Notes

  • ARIA Roles: Use role="tablist" on container, role="tab" on buttons, role="tabpanel" on content
  • aria-selected: Set to "true" for active tab, "false" for inactive tabs
  • aria-controls: Tab button must reference its panel via ID
  • aria-hidden: Set to "false" for visible panel, "true" for hidden panels
  • tabindex: Active tab has tabindex="0", inactive tabs have tabindex="-1"
  • Keyboard Navigation:
    • Tab: Moves focus into/out of tab list
    • Arrow Left/Right: Navigate between tabs
    • Home: Jump to first tab
    • End: Jump to last tab
  • Focus Visible: Purple outline appears on keyboard focus
  • Labels: Provide aria-label on tablist to describe the group

Design System Specs

Tab Height 40px
Horizontal Padding 24px (0 24px 1px 24px for inactive state)
Icon-Label Gap 4px
Inactive Text var(--text-subtle) / #4c4b53
Active Text var(--text-primary) / #5746b2 (purple)
Active Indicator 2px solid var(--text-primary) / #5746b2
Tab List Border 1px solid var(--border-medium) / #c7c6cd
Font 14px/600
Inactive Border None (padding-bottom: 1px to compensate)
Hover/Active Border 2px solid var(--text-primary) / #5746b2 (padding-bottom: 0)
Transition 150ms ease on color and border
Panel Animation 200ms fade-in on activation
