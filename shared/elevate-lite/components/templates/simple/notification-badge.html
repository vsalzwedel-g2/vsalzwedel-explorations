Usage Guidelines

  • Priority Selection: Top (red) for urgent/action-required, mid (purple) for important updates, low for informational counts
  • Count vs. Dot: Use counts when specific number matters (messages, items), dots when presence is enough (new/unread indicator)
  • Number Capping: Display "99+" for counts over 99 to maintain badge size and readability
  • Positioning: Typically positioned in top-right corner of parent element using absolute positioning
  • Clear on Action: Remove or update badge when user views/addresses the notification
  • Don't Overuse: Too many badges reduces their effectiveness — reserve for genuinely important updates

Accessibility Notes

  • ARIA Label: Add aria-label describing the notification (e.g., "3 unread messages")
  • Live Region: When count updates, use aria-live="polite" to announce changes to screen readers
  • Visible Text: Count numbers are visible and readable, no additional description needed for sighted users
  • Color Alone: Don't rely solely on red/purple/subtle colors — ensure context provides meaning (position, accompanying text)
  • Focus Management: When badge indicates actionable items, ensure clicking parent element moves focus appropriately
